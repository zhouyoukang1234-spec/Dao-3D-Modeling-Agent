# 根因诊断 — 为什么之前的 SR6 装配总是"装不起来"

> 结论先行:**问题不在 IK,不在渲染,不在 agent 能力。问题在于"装配"这一层从未真正存在过。**
> 之前的所有版本,本质上都是把"散件的网格"按一套**人手调出来的绝对坐标**摆到一个**正确的运动学骨架**旁边——
> 看起来像,但零件之间没有任何**真实的配合关系(mate)**在约束它们。骨架对了,零件却是"贴"上去的,不是"装"上去的。

## 一、先厘清:什么是对的(不要再推翻它)

逆运动学(IK)是对的。固件 `SetMainServo`/`SetPitchServo` 已被完整还原,`main@home = 0 us` 中位自洽:

- 主臂 `mainArm = 50`,主连杆 `mainRod = 175`
- 俯仰臂 `pitchArm = 75`,俯仰连杆 `pitchRod = 175`,上枢轴偏置 `55mm @ 15°`
- 这套骨架被**真实 STL 几何独立证实**(见下表),所以骨架层不是病灶。

| 零件 | 固件常数 | perceive.py 从网格实测 |
|---|---|---|
| Arm.stl | mainArm = 50 | 孔心距 **50.0** ✅ |
| L/RPitcher.stl | pitchArm = 75 | 孔心距 **75.0** ✅ |
| MainLink_Alpha.stl | mainRod = 175 | 孔心距 **175.0** ✅ |
| Receiver.stl | 偏置 55 | 枢轴距 **55.0** ✅ |

## 二、真正的病灶(architectural,不是 execution)

### 病灶 1 — 没有"配合(mate)"这一层,只有"绝对摆放(absolute transform)"
人装东西的方式是**关系式**的:"这根杆的球头落进那个球座""这个孔套上那个轴""这个面贴死那个面"。
零件的位姿是这些**约束求解出来的结果**,不是事先写死的坐标。

之前的代码(`ORS6_Stewart`)走的是反过来的路:先用 IK 算出骨架上的点 → 再把 `Arm.stl`、`MainLink.stl`
用 `ARM_PIVOT_STL` 之类**逐件手调的偏移**怼到那些点上。这意味着:
- 每个零件的摆放都是**一次性、不可推广**的 magic offset;
- 没有任何东西保证"杆的孔"真的对上"臂的孔"——它们只是**各自**贴近了骨架点;
- 任何一个偏移差几毫米/差一个朝向,整条闭环就对不上,于是"怎么操作都组装不起来"。

> 它做的是 *“把月亮画在该在的位置”*,而不是 *“让指头真的够到月亮”*。配合关系缺席,装配就只能靠运气对齐。

### 病灶 2 — 零件**变体陷阱**(同名不同尺寸,静默破坏闭环)
perceive.py 立刻抓到一个之前没人注意的硬伤:**连杆有两个版本,长度不同**。

- `MainLink_Alpha.stl`(橡胶垫圈版):孔心距 **175** —— 与固件一致;
- `BearingMainLink.stl`(轴承/杆端版):孔心距 **135** —— **比固件短 40mm**。

如果装配里混用了 135 的杆去配 175 的骨架,**闭环数学上永远无法闭合**,且没有任何报错——
零件就是"差一点点也装不上"。这正是"明明零件都在、就是装不起来"的典型现场。

### 病灶 3 — 把"感知零件"想当然了
人看一眼就知道孔在哪、轴朝哪、哪个面是装配面。算法不会。
- `probe_features.py`(朴素全壁面圆拟合)在所有零件上检出 **0 个特征**——低多边形(424–4053 顶点)、
  孔是粗八边形,朴素拟合直接失效。
- 之前的架构默认"几何特征可以随手拿到",于是跳过了**感知**,直接进入摆放。地基是空的。

### 病灶 4 — 控制/建模分离被当成"建模不需要严谨"
`FIRST_PRINCIPLES.md` 正确地把控制与建模分开,但副作用是:建模侧没有一个**可证伪的几何契约**。
没有"装配必须闭环、零件不能互相穿插、孔轴必须共线"这类**硬验证**,错误就无法在生成时被发现,
只能等到渲染出来用肉眼挑刺——于是退化成"出图→被否→再出图"的循环。

## 三、由病灶反推架构必须具备的东西(道之动:反者道之动)

1. **感知先行**:从原始网格稳健地抽取配合特征(孔/轴/面/对称),低多边形也要抽得到。→ `uam/perceive.py` 已验证可行。
2. **配合即一等公民**:零件之间用 mate(球副/转动副/圆柱副/平面贴合/紧固)来表达关系,位姿是**解**出来的。
3. **变体/单位/朝向都要被校验**:135≠175 这种陷阱必须在装配前被自动拦截。
4. **闭环与干涉是硬验证**:并联机构必须在整个工作空间闭合且不自穿插,才算"装上了"。
5. **骨架做驱动,不做摆放**:固件 IK 只负责驱动被动自由度,不负责把网格摆到坐标上。
