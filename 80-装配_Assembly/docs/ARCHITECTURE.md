# UAM — 通用装配建模架构 (Universal Assembly Modeling)

> 目标:让 AI 像人一样,把一堆零件"装"成一个能动的复杂装配体——不是把网格摆到写死的坐标旁边,
> 而是通过**感知配合特征 → 表达配合关系 → 求解位姿 → 验证闭环**,让正确的装配**自己解出来**。
> SR6(6 自由度并联机构)是第一个实践锚点。

## 0. 第一性原理:人为什么觉得"搭积木"很简单

人脑不是靠绝对坐标装配的。神经科学里的**预测编码(predictive coding)**给了一个干净的说法:

- 大脑对"这两个零件该怎么配合"先生成一个**先验预测**(这个轴该插进那个孔、这个面该贴那个面);
- 再用感官证据去**纠正预测误差**(没对齐 → 误差信号 → 调整位姿),直到误差最小;
- **可供性(affordance)**:人看到孔就"知道"它能被轴插入——这是把几何特征直接映射到配合动作的先验。

把它工程化,装配就是一个**生成—验证**的闭环:
**先验(配合意图)→ 预测(零件位姿)→ 误差(约束残差/干涉)→ 修正(求解)**。
这正好对应下面五层。`损之又损`:把"逐件手调坐标"这种多余动作不断减掉,直到只剩"声明关系、让它自解"。

## 1. 五层架构

```
 ┌──────────────────────────────────────────────────────────────┐
 │ L5 验证 Validation   闭环/干涉/可达/固件↔刚体一致性 → 可证伪硬指标 │
 │ L4 运动学 Kinematics 被动DOF + 固件IK驱动 → 全工作空间闭环        │
 │ L3 求解 Solver       满足所有 mate 的位姿解(含闭环回路)          │
 │ L2 配合 Mates        零件间关系图:球副/转动副/圆柱副/平面/紧固     │
 │ L1 语义 Affordance   特征→功能:螺孔/球座/铰轴/装配面/轴承位        │
 │ L½ 基准 Datum        从权威取回"绝对摆放",绝不臆造(本架构的心脏)  │
 │ L0 感知 Perception   原始网格→结构化特征(孔/轴/面/对称),低模也行  │
 └──────────────────────────────────────────────────────────────┘
        每个零件暴露"命名坐标系(mate connector)",装配只连接它们。
        L½ 提供它们的世界锚点——零件文件只有相对几何,绝对摆放必须取回。
```

### L0 感知 Perception  —— 状态: ✅ 已验证 (`uam/perceive.py`)
原始三角网格 → 结构化几何特征。对低多边形稳健(光滑面片连通域 + 圆柱拟合 + 内/外法向判孔/凸台)。
已在 11 个真实 SR6 STL 上抽出孔(轴/半径/中心)、凸台、孔心距,并**反推出 50/75/175/55 全套骨架常数**。

### L½ 基准 Datum  —— 状态: ✅ 已验证 (`uam/datum.py`)
零件 STL 只固定**相对几何**(轴/孔/特征间距),不含**绝对摆放**(整机装好后
身处世界何处、朝向如何)。人瞥一眼装好的样子免费获得它;agent 必须**显式从权威
取回**,绝不臆造。权威优先级(高→低):① 控制器 IK home 常数(精确、无需世界坐标)
② 装配体 CAD(STEP/F3D)③ 官方建造手册 ④ 实拍图。摆放由 Kabsch **解出**(≥3 非共线
特征,感知↔权威),残差即诚实的预测误差,如实上报、绝不调零。四条铁律(P1 自由度必须
物理 / P2 不从打印件感知作动器 / P3 控制律即基准 / P4 未约束自由度由权威闭合)硬编码为
守卫。**这是解释"为什么 8 次都装不起来"的缺失层**:旧版把绝对摆放臆造/手填,误差最多 53mm。

### L1 语义 Affordance —— 状态: ⏳ 进行中
把 L0 的裸特征分类成**功能可供性**:M3/M4 螺孔、杆端球座、608ZZ 轴承位、舵机轴、装配贴合面、对称对。
依据:半径分桶(M3≈1.7、M4≈2.2、轴承≈19)、成对/对称、与零件类型先验。

### L2 配合 Mates —— 状态: ⏳ 设计完成,待实现
每个零件输出若干**命名 mate connector**(原点+坐标系,落在特征上)。装配 = 一张图:
节点是零件实例,边是 mate(类型 + 两端 connector)。mate 类型:
`fastened(0DOF) / revolute(1R) / cylindrical(1R1T) / spherical(3R) / planar(2T1R) / parallel`。
SR6 的一条腿:`servo_horn --revolute--> Arm --spherical--> Rod --spherical--> Receiver_pivot`。

### L3 求解 Solver —— 状态: ⏳ 设计完成,待实现
给定 mate 图,解每个零件的 6-DOF 位姿使所有约束残差→0。开链直接前向传播;
并联**闭环回路**(SR6 是 6 条腿共用一个 receiver)需全局非线性最小二乘(scipy least_squares),
以固件 IK 的解作为**初值/先验**(预测编码里的 prior),大幅收敛。剩余未约束的 DOF 即机构关节。

### L4 运动学 Kinematics —— 状态: ✅ 骨架已就绪 (`anchors/sr6/constants.py`)
固件 IK 完整还原(`set_main_servo`/`set_pitch_servo`,中位自洽)。它**驱动**被动 DOF;
给定 T-Code(L0..R2)→ 6 个舵机角 → L3 在每个姿态下重解 → 得到整机随动。

### L5 验证 Validation —— 状态: ✅ 核心已就绪 (`uam/kinematics.py`)
硬指标(可证伪,取代"肉眼挑刺"):
- **固件↔刚体一致性**(已实现,通用):`consistency(home_pts, commanded_pts, R, t)`
  量化控制律命令的各铰点与单一刚体的偏离;无任何拟合自由度,残差=纯预测误差。
  SR6 实例(`anchors/sr6/rigid_consistency.py`)结论:**平移精确刚性(任意幅值残差=0,
  含耦合)**,**旋转为一阶线性化**(误差=半径 59.5 的弦减弧闭式,吻合到 4 位小数,home 处为零)。
  图见 `anchors/sr6/rigid_consistency.png`。
- **闭环**:全工作空间内每条腿残差 < ε(主子系统已验证);
- **干涉**:零件两两不穿插(trimesh 布尔/采样)——待实现;
- **变体/单位/朝向**:135≠175 之类自动拦截(L1 已具雏形);
- **对照**:与官方参考渲染(ayva_3d_ref)姿态对齐度——待实现。

## 2. 通用性主张(为什么这不只是"又一个 SR6 脚本")

L0–L3 不含任何 SR6 专有逻辑——它们只认"网格、特征、mate、约束"。SR6 只出现在 `anchors/sr6/`:
固件常数、mate 图声明、参考图。换一个装配体 = 换一个 anchor,框架不动。
这就是把"人手调坐标"`损之又损`掉之后剩下的、可迁移的内核。

## 4. 路线(道法自然,边做边长)

1. L1:特征分类 + 对称检测,把每个零件的 mate connector 自动标出来。
2. L2:声明 SR6 mate 图(一条腿先行),从 connector 实例化。
3. L3:解单腿闭环(servo→arm→rod→receiver),对照固件 IK 验证球头落位。
4. L3:扩到六腿全局解 + receiver 位姿。
5. L5:闭环/干涉/对照参考图,出可证伪指标。
6. 抽离 SR6 专有部分,固化 L0–L3 为通用内核。
